# ChanAI Pulse 思路与待办

**目录名称：** `docs/ideas_and_todos/`  
**建立日期：** 2026-07-17  
**用途：** 持续记录项目讨论中形成的思路、决策、开放问题和后续待办，为后续工程修改、科研路线设计与人工审阅提供上下文。

## 1. 本目录的定位

本目录是“持续讨论与决策工作台”，不直接替代以下正式基线文档：

- `ROADMAP.md`：正式版本路线；
- `docs/OPEN_ISSUES_AND_REFACTOR_ROADMAP.md`：已确认的工程与科学问题；
- `docs/DATASET_SPECIFICATION.md`、`docs/DATA_CONTRACTS.md`：当前数据规范；
- `docs/EXPERIMENT_DATA_PROTOCOL.md`：当前实验数据切分与增强规范；
- `docs/GUI_MANUAL_TEST_CHECKLIST.md`：人工 GUI 验收规则。

当本目录中的某项思路完成讨论并获得确认后，再将其同步到对应的正式规范、路线图或实现 PR 中。

## 3. 状态标记

- **已确认**：用户已经明确认可，应进入后续总思路；
- **建议方案**：当前推荐，但仍可继续讨论；
- **待决定**：需要用户、导师或实验结果作出选择；
- **待实现**：方向已明确，尚未进入工程实现；
- **暂缓**：保留记录，但不进入当前工作包；
- **已完成**：已经实现并完成相应验收。

## 4. 更新原则

1. 每次更新注明日期，并尽量说明是“讨论结论”还是“工程事实”。
2. 不把尚未验证的研究设想写成平台已经具备的能力。
3. 科学正确性问题与前端/重构问题分开记录。
4. 不在文档中保存真实测量数据、账号凭据、令牌或隐私信息。
5. 一个问题只指定一个主文档，其余文档通过链接或 ID 引用，避免多处状态不一致。

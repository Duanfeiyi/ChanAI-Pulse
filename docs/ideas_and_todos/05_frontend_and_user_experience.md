# 前端界面与用户体验思路

**最后更新：** 2026-07-17  
**状态：** 当前以前端稳定和人工验收为主；Base Model、快速适配与复数 H 预测交互方向已确认，完整路线见 `07_confirmed_master_plan.md`。

## 1. 当前不变的前端验收基线

- App 能从项目根目录启动；
- 三个 Tab 均能显示；
- 中英文切换不造成严重遮挡；
- 合法数据可加载，错误信息可理解；
- Characterization、Generation、Prediction 基本流程可运行；
- 窗口缩放后布局基本可用；
- 任何 App 行为变化都需要人工 GUI 验收；
- 前端问题与科学逻辑问题分开处理。

具体操作继续以 `docs/GUI_MANUAL_TEST_CHECKLIST.md` 为正式清单。

## 2. 第一页后续方向：数据与特性

建议新增或强化：

- 数据类型：H、CIR、CTF、PDP、DPSD、SAGE；
- 数据能力标签：Complex-H 可用、Power-only、Temporal、Spatial、Static；
- 维度、天线、子载波、载频、带宽、采样间隔摘要；
- 相位连续性和时间顺序检查结果；
- 幅度/相位、PDP、DS、Doppler、天线相关性等按数据能力显示；
- 缺失元数据以 WARNING 展示，不应伪造默认物理含义。

## 3. 第二页后续方向：生成与扩充

建议把“生成”和“是否加入训练”拆成两个明确动作：

```text
选择生成器
→ 生成与预览
→ 比较真实训练段与仿真数据
→ 用户决定是否加入适配训练
```

生成器选择：

- None；
- 6GPCM-lite（Lightweight Baseline）；
- QuaDRiGa（Research Engine）。

界面必须显示：生成数据来源、随机种子、场景参数、快拍数量、复数/功率表示，以及“仅加入 Train”的提示。

## 4. 第三页后续方向：模型、适配与预测

建议按以下区域组织：

1. Base Model：模型 ID、版本、适用范围、输入维度；
2. Prediction Mode：直接预测、快速适配、深度微调；
3. Data Split：适配训练、验证、独立测试的时间范围；
4. Adaptation Status：冻结层、可训练参数量、epoch、Validation；
5. Comparison：Base 与 Adapted 的相同指标对比；
6. Future Prediction：预测步数、时间跨度、输出域；
7. Save / Export：保存 Adapted Model、实验配置和结果。

## 5. 用户默认路径

普通用户应能以较少决策完成：

```text
导入数据
→ 查看质量检查与可视化
→ 接受平台推荐的 Base Model
→ 选择“快速适配”
→ 可选开启数据扩充
→ 运行适配
→ 查看适配是否优于直接预测
→ 预测未来 H
```

高级参数放入折叠区域，避免普通用户必须理解所有学习率、冻结层和生成器细节。

## 6. 状态、错误和回退

- 训练、适配和 QuaDRiGa 生成显示清晰进度；
- 支持取消长任务；
- 数据不足时解释为什么只能直接预测；
- 相位不连续时解释为什么不能进入复数 H 管线；
- 模型维度不兼容时推荐可用模型或转换方案；
- 适配变差时显示“已回退 Base Model”；
- 失败后恢复按钮、下拉框和原模型状态；
- 不让用户误以为训练损失就是最终 Test 性能。

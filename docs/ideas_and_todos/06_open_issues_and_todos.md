## 7. 建议总顺序

```text
完成当前 App 模块化拆分和人工验收
→ 修正文档基线与关键科学问题
→ 确定第一版复数 H 任务和数据契约
→ 建立 QuaDRiGa / 开放数据管线
→ 完成动态宽带 SISO 复数预测基线
→ 训练通用 Base Model
→ 实现用户数据快速适配
→ 建立复现实验与 Benchmark
→ 扩展固定规模 MIMO
→ 再研究跨场景、跨频段和更复杂模型
→ 最后进行完整 UI 集成与长期平台扩展
```

## 8. 暂缓事项

- 前端美化和非关键字体/图例微调；
- Web / Cloud 迁移；
- 任意天线规模的通用 MIMO；
- 复杂值网络、Transformer 和大规模架构竞赛；
- 未经 Benchmark 的模型替换；
- 未经许可的真实数据公开；
- 用“全频段全场景预测”描述尚未完成验证的当前能力。
